# ТЗ: Локальная авторизация без SSO для mh-calc

**Дата:** 2026-06-20
**Статус:** Реализовано (Гейт 4 пройден). Область: только dev/локально, прод на SSO.
Локальная БД переведена на MySQL — миграции проекта написаны под MySQL и на свежей
SQLite взаимно конфликтуют (multiple dropColumn vs ->change()).

## Проблема

Калькулятор (`mh-calc`) не имеет собственной авторизации. Единственный вход —
`POST /api/v1/login` с `sso_token`, который проверяется на внешнем главном сайте
Marine Health (`ApiOutMH` → `/api/sso/authenticate`). Локально главного сайта нет
(`MAIN_PROJECT=:4000` не поднят), поэтому фронт при заходе на `:3000` сразу
редиректит на `:4000` → `ERR_CONNECTION_REFUSED`. Открыть калькулятор локально нельзя.

## Цель

Дать возможность **создать аккаунт и войти по email+паролю без SSO**, чтобы
калькулятор работал автономно в локальной/dev-среде. Продакшн-флоу через SSO
остаётся без изменений.

## Пользовательские сценарии

1. Локальный разработчик открывает калькулятор на `:3000` и вместо редиректа на
   `:4000` видит экран входа.
2. Может зарегистрировать аккаунт: email + пароль (+ опц. имя, язык, валюта).
3. Может войти по email + паролю и попасть в калькулятор — ему выдаётся
   `CalculatorUserToken`, который сохраняется как `userToken` (как в SSO-флоу).
4. Весь путь локальной авторизации **отключён в проде** (gated) — прод остаётся на SSO.

## Ограничения

- Backend: Laravel 10, модуль `Calculator`. Переиспользуем `CalculatorUser` и
  `CalculatorUserToken`. Добавляем nullable-колонку `password` в `calculator_users`.
- Новые эндпоинты под `/api/v1/auth`: `register`, `login` (локальные). Ответ — той же
  формы, что у SSO-логина (`{status, token, email, profile}`), чтобы переиспользовать
  обработку токена на фронте.
- Gating backend: локальные эндпоинты недоступны, если не включены (конфиг-флаг,
  по умолчанию включён при `APP_ENV != production`).
- Frontend: Next.js. Форма входа/регистрации вместо редиректа на `MAIN_PROJECT`,
  показывается только когда включён флаг (`NEXT_PUBLIC_LOCAL_AUTH=true`). Переиспользуем
  существующие antd-компоненты и `widgets/form`.
- Откатить временную dev-token заглушку (`.env.local NEXT_PUBLIC_DEV_USER_TOKEN`,
  `utils.js DEV_USER_TOKEN`, `GlobalContext.js` dev-ветка).
- Секреты: пароли — bcrypt (`Hash::make`), в БД только хеш.

## Границы (НЕ входит в эту итерацию)

- Сброс пароля, подтверждение email, "remember me", refresh-токены.
- Управление пользователями, роли, админка.
- Любые изменения продакшн-SSO-флоу.

## Тесты (из требований)

- register: создаёт пользователя + токен; повтор email → ошибка; слабый пароль → ошибка.
- login: верные креды → токен; неверные → 401; несуществующий email → 401.
- gating: при выключенном флаге эндпоинты → 404/403.
- middleware: выданный токен проходит `calculator.validate.token`.
