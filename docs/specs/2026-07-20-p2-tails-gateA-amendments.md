## Обязательные амендменты (must-fix ревью)

### A-t1 (security, high/high): прокси-секрет ОБЯЗАТЕЛЕН
`AuthController::telegramLogin` выдаёт admin-токен **только** при валидном заголовке
`X-Admin-Proxy-Key` (сравнение `hash_equals`, значение из env/KV). Заголовок ставит только
Next BFF server-side. Без этого XSS на admin-домене мог перехватить payload Login Widget,
дёрнуть публичный `/api/v1/auth/telegram-login` напрямую (CORS `*`) и прочитать токен —
httpOnly-дизайн обходился бы. Закрывает «Развилку №2» плана: вариант «оставить Bearer-приём
как есть» отклонён.

### A-t3 (correctness, high/medium): `migrate:fresh` только при явном `--fresh`
Противоречие плана устранено: деструктивный `migrate:fresh` исполняется **только** при явно
переданном `--fresh`; guard'ы (`environment('production')` → отказ, allowlist имён БД,
непустая `members` без `--fresh` → отказ) проверяются **до** любой деструктивной операции.
Без `--fresh` и с непустой БД — отказ с подсказкой.
