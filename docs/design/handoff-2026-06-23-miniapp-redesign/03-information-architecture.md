# 03 · Информационная архитектура и навигация

## Глобальная структура

```
Telegram (системный хедер: ‹ назад / × закрыть, заголовок)
└── IziGo Mini App
    ├── [tab] Доход        (wallet)      — всегда
    ├── [tab] Магазин      (shop)        — всегда
    ├── [tab] Команда      (team)        — всегда
    ├── [tab] Ранг         (rank)        — всегда
    ├── [tab] Профиль      (profile)     — всегда
    ├── [tab] Уведомления  (inbox)       — по флагу c1_notifications
    └── [tab] Поддержка    (helpdesk)    — по флагу c2_helpdesk
```

Нижний таб-бар — постоянный навигатор между корневыми вкладками. Активная вкладка — акцентный
цвет; неактивная — приглушённый. Иконка над подписью. Высота ≈ 60–64px + safe-area.

> **Важно для дизайна таб-бара:** вкладок может быть 5, 6 или 7. Спроектировать так, чтобы
> 7 иконок с подписями помещались на узком экране (≈320px) без переполнения — например,
> компактные подписи, меньший трекинг или адаптивная ширина ячеек.

## Карта экранов и модальных потоков

### Вкладка «Доход» (wallet)
- Hero-баланс: всего начислено + доступно к выводу.
- Кнопки **Вывести** и **Пополнить**.
- Бонусы по 4 типам (сетка).
- Текущий пакет (или приглашение купить).
- Последние начисления (список).
- Мои заявки на вывод (если есть).
- История операций (если есть) → ссылка **Выписка**.
- → модалка **Вывод средств** (сумма + TON-адрес).
- → модалка **Пополнение баланса** (сумма) → чек-аут TON Pay.
- → модалка **Выписка за период** (диапазон дат, итоги, CSV-экспорт).

### Вкладка «Магазин» (shop)
Сегмент-переключатель: **Каталог / Мои заказы / Автозаказ**.
- Каталог: карточки товаров → **Купить** → чек-аут TON Pay.
- Мои заказы: список заказов со статусами, трек-номерами; неоплаченный → **Оплатить**.
- Автозаказ: список подписок (пауза/возобновить/отменить) → **+ Новый автозаказ** (модалка).
- → **TON Pay чек-аут** (общий поток оплаты, см. ниже).

### Вкладка «Команда» (team)
- Сводка: в команде / активных / личных.
- Фильтр: Все / Активные / Новые.
- Дерево команды (вложенные узлы с отступом по уровням, аватар + имя + статус).

### Вкладка «Ранг» (rank)
- Текущий ранг → следующий.
- Прогресс по «малой ветке PV» (прогресс-бар).
- Прогресс по числу приглашённых (прогресс-бар).
- Что открывает следующий ранг (инфо-блок).

### Вкладка «Профиль» (profile)
- Идентичность: аватар, имя, статус, ранг, ключевые цифры (заработано / приглашено / в команде / ID).
- Реферальная ссылка: код + ссылка + **Копировать** + **Поделиться** (Telegram share).
- KYC-верификация: статус + **Пройти верификацию**.
- Соглашение: статус версии + **Принять**.
- Совладельцы/Наследники (по флагу c6_copartners) — справочный список + CRUD-модалка.
- Настройки: ранг, статус, **Закрыть**.

### Вкладка «Уведомления» (inbox, по флагу)
- Список уведомлений (непрочитанные выделены), **Прочитать всё**, тап → прочитано.

### Вкладка «Поддержка» (helpdesk, по флагу)
- Список тикетов со статусами ↔ чат-тред (сообщения пузырями), **Новое обращение**, ответ.

### Глобальные/онбординг-потоки
- **Заглушка вне Telegram:** «Откройте через Telegram».
- **Гейт соглашения:** немодально-закрываемая модалка с текстом соглашения + **Принимаю**
  (показывается до принятия, блокирует работу).
- **TON Pay чек-аут** (общий): адрес получателя + memo (копируемые) + кнопка кошелька +
  «Проверить оплату» + статус (ожидание/подтверждено/не подтверждено).
- **Глобальные состояния:** Loading (спиннер), Error («Ошибка загрузки» + Повторить),
  Auth-error («Откройте через Telegram»).

### Калькулятор (опционально, отдельная поверхность)
Публичный симулятор бинарного плана на дереве (`react-d3-tree`): построение узлов, смена пакета,
зум, шеринг ссылки, расчёт бонусов по узлу. Не часть кабинета; редизайн — по желанию (см.
`04-screens-spec.md`).
