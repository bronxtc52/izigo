# Handoff: редизайн мобильного приложения IziGo (Telegram Mini App)

**Дата:** 2026-06-23
**Цель:** полный дизайн-handoff + ТЗ для создания **нового визуального дизайна** мобильного
приложения IziGo через **Claude Design** (или любой дизайн-инструмент/дизайнера).
**Тип:** партнёрский кабинет MLM-платформы внутри Telegram Mini App.

> Это пакет ввода для дизайнера. Он описывает **что** за продукт, **какие экраны и данные**
> в нём есть и **какие рамки нельзя нарушать**. Сам визуальный язык (цвета, типографика,
> ритм, компоненты) — предмет редизайна; старые токены даны лишь как точка отсчёта, их можно
> и нужно переосмыслить, не ломая жёсткие ограничения (см. `02-constraints.md`).

---

## Как пользоваться этим handoff с Claude Design

1. Прочитать `01-product-brief.md` — что такое IziGo, кто пользователь, цели редизайна, тон.
2. Усвоить `02-constraints.md` — **жёсткие нерушимые рамки** (Telegram-контейнер, темы,
   валюта, навигация, локали). Их нарушение = брак.
3. Использовать `03-information-architecture.md` + `04-screens-spec.md` как карту экранов и
   контента: что рендерится, какие поля данных, какие состояния (loading/empty/error).
4. `05-design-system-reference.md` — текущие токены и компоненты как **baseline для эволюции**,
   не как обязательную цель.
5. `06-creative-brief.md` — готовый **бриф/промпт**, который можно подать в Claude Design как
   постановку задачи на дизайн. Это «ТЗ на дизайн» в сжатом, исполняемом виде.

## Контекст одной строкой

IziGo — товарный MLM (бинарный маркетинг-план) внутри Telegram. Партнёр покупает пакет,
строит команду, получает 4 типа бонусов (Бинар / Реферал / Лидер / Ранг) в USDT-кошелёк и
выводит средства в USDT на TON-адрес. Авторизация — только через Telegram, валюта — только
USD/USDT, оплата — только TON Pay. Всё происходит на узком мобильном экране внутри Telegram.

## Что НЕ входит в редизайн

- **Веб-админка** (`admin.izigo.adarasoft.com`) — отдельный продукт, не Mini App. Не трогаем.
- **Калькулятор** (публичный симулятор маркетинг-плана на `react-d3-tree`) — отдельная
  поверхность. В этом редизайне опционален (см. `04-screens-spec.md` §«Калькулятор»);
  основной фокус — партнёрский кабинет Mini App.
- **Бэкенд, движок расчёта бонусов, формулы** — не меняются. Дизайн только потребляет выходы.
