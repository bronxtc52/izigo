# 02 · Жёсткие ограничения (нерушимые)

Это рамки продукта и платформы. Их нарушение делает дизайн неприменимым. Всё, что **не**
перечислено здесь, — свободное поле для редизайна.

## Платформа и контейнер

- **Telegram Mini App.** Приложение открывается внутри Telegram (WebView), а не как
  самостоятельный сайт/нативное приложение. Сверху — системный хедер Telegram (назад/закрыть,
  заголовок), его не перерисовать. Дизайн живёт в области контента под ним.
- **Узкий мобильный экран.** Проектировать под телефон, ширина контента ≈ 320–430px. Никаких
  десктоп-раскладок, сайдбаров,多колоночных сеток. Вертикальный скролл — основной паттерн.
- **Safe-area.** Учитывать нижнюю safe-area (домашний индикатор iOS) и системный хедер Telegram.
- **Только Telegram-вход.** Авторизация исключительно через Telegram `initData` (никаких
  экранов логина/пароля, email-форм, OTP, 2FA). Вне Telegram приложение показывает заглушку
  «Откройте через Telegram». Экранов регистрации/входа проектировать **не нужно**.

## Темизация

- **Светлая и тёмная темы обязательны и равноправны.** Тема приходит от Telegram
  (`Telegram.WebApp.colorScheme` + `themeParams`) и может меняться на лету. Дизайн поставляется
  **в обоих вариантах** для каждого экрана.
- Цвета привязываются к Telegram `themeParams` (bg_color, text_color, hint_color, button_color,
  secondary_bg_color, section_separator_color, destructive_text_color). Но если тема Telegram
  даёт недостаточный контраст для критичного элемента (например primary-кнопки) — допускается
  **фиксированный фирменный акцент IziGo** вместо подменённого Telegram-цвета.
- **Контраст WCAG AA** обязателен в обеих темах: основной текст ≥ 7:1, вторичный ≥ 4.5:1,
  бейджи ≥ 4.5:1.

## Деньги и оплата

- **Только USD/USDT.** Все суммы — в долларах (USDT ≈ USD 1:1). Никаких мультивалютных
  переключателей, символов других валют, конвертаций в кабинете. Формат: `$1 234.50`.
- **Цифры — табличные (tabular figures)** для корректного выравнивания сумм по разрядам.
- **Оплата — только TON Pay** (USDT-перевод на TON). Никаких карт/банков/иных шлюзов в UI
  Mini App. Чек-аут показывает адрес получателя + обязательный memo + подтверждение через кошелёк.
- **Вывод — только USDT на TON-адрес.** Поле адреса — критичное и необратимое, требует явного
  предупреждения «ошибка адреса необратима».

## Навигация

- **Нижний таб-бар** — основной навигатор. **5 постоянных вкладок**:
  `Доход · Магазин · Команда · Ранг · Профиль`.
- **До 2 дополнительных вкладок** появляются по feature-флагам (могут быть скрыты):
  `Уведомления`, `Поддержка`. Дизайн должен корректно выглядеть и с 5, и с 6, и с 7 вкладками
  (предусмотреть компактный режим таб-бара при 6–7 иконках на узком экране).
- **Админки в Mini App нет** — она вынесена в отдельный веб-продукт. Не проектировать.

## Локализация

- **6 локалей:** kk (казахский, по умолчанию), ru, mn, uz, ky, az. Текст переменной длины —
  не фиксировать ширину под одну строку, закладывать перенос/усечение по границам слов.
- Числа/суммы — единый формат во всех локалях (USD).

## Продуктовая суть (не переопределять смысл)

- **Бинарный** маркетинг-план: у партнёра левая и правая ветка. Команда — дерево.
- **4 типа бонусов:** Бинар (binary), Реферал (referral), Лидер (leader), Ранг (rank). Каждому
  закреплён семантический цвет (см. `05-design-system-reference.md`) — менять можно палитру, но
  **4 типа должны оставаться визуально различимыми**.
- **Пакеты:** Bronze ($100 / 90 PV), Silver ($200 / 180 PV), Gold ($600 / 540 PV). Активация
  пакета — только через покупку в Магазине.
- **Ранги** с прогрессией: текущий → следующий, прогресс по условиям (объём малой ветки PV +
  число приглашённых).

## Что вне scope

- Экраны входа/регистрации (вход = Telegram).
- Веб-админка.
- Любые платёжные методы кроме TON Pay.
- Мультивалютность.
- Десктоп/планшет-раскладки.
