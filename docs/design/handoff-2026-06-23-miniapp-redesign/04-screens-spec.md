# 04 · Поэкранная спецификация

Контент, реальные поля данных (из прод-API), действия и состояния каждого экрана. Все строки —
текущие русские строки продукта (новый дизайн может уточнить копирайт, смысл сохранить). Суммы —
USD, формат `$1 234.50`.

Легенда состояний: **Loading** (загрузка), **Empty** (нет данных), **Error** (ошибка),
**Auth-error** (вне Telegram → «Откройте через Telegram»).

---

## 1 · Доход (wallet) — главный экран

**Назначение:** обзор заработка, вывод и пополнение, разбивка бонусов, история.

**Блоки:**

1. **Hero-баланс** (главный акцент экрана)
   - «Всего начислено» → `$dash.total` — самая крупная сумма (hero).
   - «Доступно к выводу» → `$wallet.available`.
   - Если `wallet.held > 0`: «в холде $…».
   - Если `wallet.clawback_debt > 0`: «к компенсации $…» (цвет ошибки).
   - Кнопка **Вывести** (disabled, если available ≤ 0) и **Пополнить**.

2. **Бонусы по типам** — сетка из 4 ячеек по `dash.by_type`:
   `binary`→**Бинар**, `referral`→**Реферал**, `leader`→**Лидер**, `rank`→**Ранг**.
   Каждая: цветная точка (цвет типа) + название + сумма `$…`.

3. **Пакет**
   - Если активирован: «Текущий: **<название пакета>**» + кнопка **Сменить в Магазине**.
   - Если нет: «Пакет не активирован» + кнопка **Купить в Магазине**.

4. **Последние начисления** — список `dash.lines`: точка типа + бейдж типа + `+$amount` (зелёный).
   Empty: «Пока пусто».

5. **Мои заявки** (если есть заявки на вывод) — список: `$amount` + статус-бейдж + дата.
   Статусы: requested→«на рассмотрении», approved→«одобрена», paid→«выплачена»,
   rejected→«отклонена», cancelled→«отменена».

6. **История операций** (если есть) + ссылка **Выписка**.
   Строка: тип (accrual→«Начисление», withdrawal→«Вывод») + дата, `$amount` (списания — серым/минус).

**Модалки этого экрана:**
- **Вывод средств:** «Доступно: $…»; поле суммы (мин 0.01, префикс $); поле «TON-адрес для USDT
  (например EQ…)»; подсказка «Выплата приходит в USDT на указанный TON-адрес. Проверьте адрес —
  ошибка необратима» (выделить как опасное); кнопка подтверждения.
- **Пополнение баланса:** подсказка «Пополнение USDT через TON Pay. Баланс используется для
  автозаказов и покупок»; поле суммы → ведёт в TON Pay чек-аут.
- **Выписка за период:** выбор диапазона дат; кнопка **Показать**; кнопка **CSV**; итоги —
  **Поступило / Списано / Итог**; список движений. Empty: «Нет движений за период».

**Данные:** `mmDashboard` → `{total, by_type:{binary,referral,leader,rank}, lines:[{type,amount}]}`;
`mmWallet` → `{available, held, clawback_debt}`; `mmWithdrawals` → `[{amount,status,requested_at}]`;
`mmWalletTx` → `{items:[{amount,source_type,source_id,created_at}]}`;
`mmWalletStatement` → `{summary:{credited_cents,debited_cents,net_cents}, items:[…]}`.

## 2 · Магазин (shop)

**Назначение:** покупка пакетов/товаров за USDT, заказы, автозаказ.
Сегмент-переключатель: **Каталог / Мои заказы / Автозаказ**.

- **Каталог** — карточки `mmCatalog` → `[{id,name,description,pv,stock,price_usdt_cents}]`:
  название, описание, бейдж `{pv} PV`, бейдж «нет в наличии» если stock ≤ 0, цена `$…`,
  кнопка **Купить** → создаёт заказ → TON Pay чек-аут. Empty: «Каталог пуст».
- **Мои заказы** — `mmOrders` → `[{id,status,items:[{name,qty}],total_pv,total_usdt_cents,created_at,tracking_no}]`:
  `#id` + товары ×кол-во + статус-бейдж, `{total_pv} PV` + дата + «трек {tracking_no}», цена `$…`.
  Если `pending_payment` → кнопка **Оплатить**. Empty: «Заказов пока нет».
  Статусы: pending_payment→«ожидает оплаты», paid→«оплачен», processing→«в обработке»,
  shipped→«отправлен», delivered→«доставлен», cancelled→«отменён», refunded→«возврат».
- **Автозаказ** — `mmAutoship` → `[{id,product_id,status,interval_days,next_charge_at,retry_stage}]`:
  товар + статус-бейдж, «каждые {N} дн. · следующее {дата} · повтор д.{N}»,
  действия **Пауза/Возобновить/Отменить**. **+ Новый автозаказ** (модалка: выбор товара,
  интервал 1–365 дней, подсказка «Списание — с внутреннего USDT-баланса. Пополните его во
  вкладке „Доход“»). Статусы: active→«активна», paused→«на паузе», cancelled→«отменена».

## 3 · Команда (team)

**Назначение:** бинарная команда деревом.

- **Сводка:** **В команде** `total` · **Активных** `active` (зелёный) · **Личных** `personal`.
- **Фильтр:** Все / Активные / Новые.
- **Дерево** `mmTree` → `{name, attributes:{id,status}, children:[…рекурсивно…]}`:
  узел = аватар (инициалы, тинт по статусу) + имя + статус-бейдж («активен»/«нов»).
  Вложенность — отступ + вертикальная линия слева по уровням. Корень — «Вы».
  Empty: «Команда пуста».

> Дизайнерская задача: компактная, читаемая визуализация **глубокого дерева** на узком экране —
> отступы по уровням, сворачивание/разворачивание ветвей, понятная связь родитель→потомки.

---

## 4 · Ранг (rank)

**Назначение:** текущий ранг и прогресс к следующему.
`mmRank` → `{current:{alias}, next:{alias, conditions:{small_branch_pv, personal_count}},
progress:{small_branch_pv, personal_count}}`.

- **Шапка:** иконка-трофей + «Текущий ранг» `current.alias` (или «нет»); справа «далее»
  `next.alias` (или «макс.»).
- Если есть следующий ранг:
  - **Малая ветка PV:** `progress.small_branch_pv / next.conditions.small_branch_pv` + прогресс-бар.
  - **Приглашённые:** `progress.personal_count / next.conditions.personal_count` + прогресс-бар.
  - **Что открывает ранг {next.alias}:** «＋ Лидерский бонус от объёма ветки», «＋ Повышенные
    условия квалификации».

## 5 · Профиль (profile)

`mmMe` → `{member:{id,name,status,package_id,ref_code,rank:{alias}}, ref_link}`.

- **Идентичность:** аватар (инициалы), имя, статус-бейдж («● Активен»/«Новый»), ранг-бейдж.
  Цифры: **Заработано** `$dash.total` · **Приглашено** `personal` · **В команде** `total` · **ID** `#id`.
- **Реферальная ссылка:** код `ref_code`; read-only поле со ссылкой; **Копировать ссылку**
  (тост «Ссылка скопирована»); **Поделиться** (открывает Telegram share).
- **Верификация (KYC):** `mmKyc` → `{status, reject_reason}`. Статус-бейдж:
  none→«не пройдена», pending→«на проверке», approved→«подтверждена», rejected→«отклонена».
  При rejected — показать причину. При none/rejected — кнопка **Пройти верификацию**. При
  pending — подсказка «Заявка на проверке. Дождитесь решения — это требуется для вывода крупных сумм».
- **Соглашение:** `mmAgreement` → `{version, accepted, text}`. «Статус (версия {version})» →
  бейдж «Принято» или кнопка **Принять**.
- **Совладельцы / Наследники** (по флагу c6_copartners): `mmCopartners` →
  `[{id,kind,full_name,phone,share_percent,note}]`. Бейдж «Наследник»/«Совладелец» + ФИО +
  «телефон · доля% · заметка». **Добавить/Изм./Удал.** Подсказка: «Справочная информация. Не
  влияет на начисления, выплаты и структуру».
- **Настройки:** Ранг, Статус, **Закрыть** (закрывает Mini App).

## 6 · Уведомления (inbox) — по флагу c1_notifications

`mmNotifications` → `[{id,title,body,created_at,read}]`. Список; непрочитанные — жирные +
подсветка фона. Поле `body` — серверный Telegram-HTML (рендерится как разметка). Заголовок
списка + **Прочитать всё**. Тап по непрочитанному → прочитано. Empty: «нет уведомлений».

---

## 7 · Поддержка (helpdesk) — по флагу c2_helpdesk

Список тикетов ↔ чат-тред (поллинг ~6с).
- **Список:** `[{id,subject,status,last_message_at}]`. Статус-бейдж: open→«Открыт» (синий),
  in_progress→«В работе» (янтарный), resolved→«Решён» (зелёный), closed→«Закрыт» (серый).
- **Новое обращение** (модалка): тема (до 160) + текст (до 4000).
- **Чат:** сообщения пузырями — мои (`author_role==='member'`) справа/акцент, поддержка слева;
  поле ответа (до 4000) + **Отправить**. Закрытый тикет → подсказка о закрытии.

---

## 8 · TON Pay чек-аут (общий поток оплаты)

Модалка «Оплата TON Pay». Фазы: `idle → sending → awaiting → paid / failed`.
Инвойс: `{payment_id, amount_cents, currency, merchant_address, memo}`.
- «К оплате» `$amount` + валюта (USDT).
- Копируемые строки: **Адрес получателя** (`merchant_address`) и **Memo (обязательно)** (`memo`)
  — memo критичен, подсветить «обязательно».
- Кнопка кошелька (TonConnect) + **Оплатить через кошелёк**.
- **Проверить оплату** (поллинг ~4с, ~90с, затем вручную).
- Успех: «Оплата подтверждена» («Заказ оплачен, тариф активирован» / «Платёж зачислён»).
- Ошибка: «Платёж не подтверждён».

> Подтверждение оплаты — **только на сервере**. UI отражает статус, не «решает» его.

---

## 9 · Глобальные состояния и гейты

- **Auth-error / вне Telegram:** «Откройте через Telegram» / «Mini App доступен только внутри
  Telegram (авторизация по initData)». (Также при заходе в браузере на /cabinet, /admin.)
- **Loading:** спиннер по центру.
- **Error:** «Ошибка загрузки» / «Не удалось получить данные. Попробуйте позже» + **Повторить**.
- **Гейт соглашения:** немодально-закрываемая модалка «Пользовательское соглашение» с
  прокручиваемым текстом + **Принимаю** (показывается, пока не принято).

## 10 · Калькулятор (опционально)

Публичный симулятор бинарного маркетинг-плана (отдельная поверхность, не часть кабинета).
Бинарное дерево на `react-d3-tree`: узлы-карточки (аватар, имя, ранг, спонсор, контракт/пакет,
левая/правая ветка с PV), добавление узла (＋ на пустой ветке), удаление, смена пакета для всей
структуры, зум, рекадрирование, шеринг view/edit-ссылок, расчёт бонусов по узлу. Шапка с KPI:
итоговая прибыль, реферальный бонус (I/II), товарооборот структуры, лидерский бонус, квалиф.
награда. **Мультиязычный + мультивалютный** (исключение из «только USD» — это симулятор, не
биллинг). Редизайн калькулятора — по желанию, низкий приоритет относительно кабинета.
