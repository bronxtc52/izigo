# Handoff: продуктовая логика «Лидер» (sdm-kz/leader-new)

> Снято с легаси-репо `sdm-kz/leader-new` («Брокерская система Лидер», SDM) — **read-only разведка
> структуры**, без технического стека. Репо заморожен (последние коммиты 2020). Цель документа —
> зафиксировать, **как устроена платформа продуктово** (вкладки/экраны/роли/процессы), чтобы
> выбрать, что адаптировать под наш `binar-mlm` (IziGo). План адаптации — в
> [`roadmap-adaptation.md`](./roadmap-adaptation.md).
>
> Жёсткие рамки нашего проекта (не нарушать при адаптации): **движок расчёта бонусов не трогаем;
> валюта только USD; платёж и авторизация только Telegram.**

---

## 1. Что это за платформа
Полноразмерный личный кабинет MLM/«брокерской» сети: участники покупают продукты/пакеты,
строят структуру (рефералы + дерево), получают начисления («mining» — их механизм дохода),
выводят деньги; администрация управляет людьми, деревом, выплатами, контентом и отчётностью.
По сути — то, чем наш проект хочет стать на уровне UX, но у нас вместо «mining» свой **движок
бонусов** (binary/referral/leader/rank).

Роли: участник (user), администрация (admin) с под-уровнями (в т.ч. SuperAdmin).

---

## 2. Кабинет участника — вкладки (`App/user/`)

| Вкладка | Продуктовый смысл |
|---|---|
| **dashboard** | Главная: сводка по доходу/структуре/статусу, точка входа в остальное. |
| **cabinet** | Корневой каркас кабинета (обёртка вкладок). |
| **finances** | Финансы: баланс, история начислений и операций, заявки на вывод. |
| **command** | «Команда»/структура: дерево/список downline. |
| **mining** | Механизм начисления дохода участнику (их аналог нашего движка бонусов). |
| **products** | Магазин: каталог, покупка/апгрейд пакета. |
| **profile** | Профиль: личные данные, реквизиты, настройки. |
| **referrals** | Рефералы: реф-ссылка, приглашённые. |
| **news** | Лента новостей/анонсов от компании. |
| **chat** + **messages** | Внутренняя переписка (канал + личные сообщения). |
| **kb** + **kb.article** | База знаний/обучение: статьи, материалы. |
| **voip** | Звонки (IP-телефония). |
| **agreement** | Юридические согласия/акцепт договоров. |
| **users** / **user** | Просмотр участников. |

Дополнительно (MVC-страницы вне SPA): **Account** — Hierarchy (генеалогия), Login/Register,
ForgotPassword/Reset, 2FA (SendCode/VerifyCode), подтверждение email, внешний вход; **Manage** —
телефон, смена пароля, привязки логинов.

## 3. Админка — экраны (`App/admin/`)

| Экран | Продуктовый смысл |
|---|---|
| **users** (admin.users / admin.user / admin.user.new) | Управление участниками: список, карточка, создание. Под-области: SuperAdmin, Changing (правки), Mining (начисления). |
| **command/tree** | **Ручной плейсмент/перенос участников в дереве.** Ключевая операционная фича. |
| **payouts** (payout / payouts) | Очередь и обработка выплат. |
| **mining** (settings / stats / payouts / payout) | Настройки механизма дохода, статистика, выплаты по нему. |
| **news** (+ item) | Управление новостями/анонсами. |
| **notifications** (+ item) | Рассылки/уведомления участникам. |
| **kb** (details / journal) | Управление базой знаний/обучением. |
| **products** (details / journal) | Управление каталогом/пакетами. |
| **reports/balances** | Отчёт по балансам участников. |
| **reports/users** | Отчёт по пользователям. |
| **subdivisions** (details / journal) | Подразделения/филиалы оргструктуры. |
| **currency/rates** | Курсы валют (мультивалютность). |

Дополнительно: десятки **email-шаблонов** (NewParticipant, NewPayout, PayoutApproved,
MiningPayoutApproved, ProductUpgrade…), **Agreement** (AcceptanceAct / System / User — тексты и
акцепт договоров), отдельный **PHP-Shop** с `reward_points` (бонусные баллы, отдельный модуль).

🟡 — отложено, 🔴 — исключено навсегда по ограничениям. Детали — в роадмапе.
